# Frontend v2 (Phase 2) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Elm frontend with TypeScript + React + D3, per the design in `docs/superpowers/specs/2026-07-11-frontend-v2-design.md`, replicating every current feature (pan/zoom map, tiles, markers, transport lines, controls, recipe toggles, stats) plus two new visualizations Phase 1 makes possible (factory cash-health coloring, a shortage panel).

**Architecture:** A new `frontend/` Vite + React + TypeScript project polls the existing Go backend's JSON API over relative URLs (proxied by Vite in dev, same-origin once built and served by the Go backend in production). D3 (`d3-zoom`) manages a single pan/zoom transform on an SVG `<g>`; React owns everything rendered inside it declaratively. Two small additive fields (`Factory.Cash`, `State.Shortages`) are added to the Go wire types to support the new visualizations.

**Tech Stack:** Go 1.21 (existing backend, additive changes only), Node.js + Vite + React 18 + TypeScript (strict) + `d3-zoom`/`d3-selection`/`d3-scale`/`d3-scale-chromatic`.

## Global Constraints

- Backend changes are additive-only to `state/http/http.go` and `state.toHTTP()` (`state/state.go`) — no other Phase 1 logic changes. Go 1.21.0, `go build ./...` and `go test ./...` must pass after Task 1.
- Every frontend task's automated gate: from `frontend/`, `npx tsc --noEmit` and `npm run build` must both succeed. No test framework is being added in this plan (per the approved design) — this is the deliberate quality gate for frontend tasks instead of unit tests.
- **Visual/interactive correctness cannot be verified by an implementer subagent alone** (no browser available in that context). Implementer subagents must reason carefully about D3/SVG correctness (coordinate flips, transform order, z-order, matching the original Elm app's exact behavior) since they cannot see the render. The controller (not a dispatched subagent) performs an explicit real-browser check after Task 7 (markers complete) and again after Task 11 (full integration) using either the user's own confirmation or the claude-in-chrome tooling.
- All API calls use **relative URLs** (`/state`, not `http://localhost:28100/state`) — Vite's dev-server proxy and same-origin production serving both depend on this. Do not hardcode a host anywhere in the frontend.
- TypeScript wire types in `frontend/src/types.ts` must match `state/http/http.go`'s JSON tags exactly, field for field.
- Follow the existing repo's commit-message style. Work directly on `main`, matching the established Phase 1 pattern (no worktree), unless told otherwise.

```go
// shortageWireLimit caps how many entries toHTTP reports, so the frontend
// panel doesn't get flooded once dozens of products have small residual
// shortages.
const shortageWireLimit = 20

func (s *State) shortagesForWire() []statehttp.Shortage {
	shortages := make([]statehttp.Shortage, 0, len(s.unmet))
	for product, amount := range s.unmet {
		shortages = append(shortages, statehttp.Shortage{Product: product, Amount: amount})
	}
	sort.Slice(shortages, func(i, j int) bool {
		return shortages[i].Amount > shortages[j].Amount
	})
	if len(shortages) > shortageWireLimit {
		shortages = shortages[:shortageWireLimit]
	}
	return shortages
}
```

### Task 3: Polling hook

**Files:**
- Create: `frontend/src/hooks/usePolling.ts`

**Interfaces:**
- Consumes: `api.getState` (Task 2).
- Produces: `usePolling(): { state: State | null, error: string | null, applyState: (s: State) => void }` — a hook that fetches `/state` on mount and, after any state update where `running` is true, re-polls after 50ms (matching the Elm app's exact `sleepAndPoll` behavior and interval). `applyState` is exported so button handlers in later tasks (Run/Tick/Stop/Reset/recipe toggle) can feed their own responses through the same continuation logic.

- [ ] **Step 1: Implement the hook**

Create `frontend/src/hooks/usePolling.ts`:

```typescript
import { useCallback, useEffect, useRef, useState } from 'react';
import { getState } from '../api';
import type { State } from '../types';

// Matches the original Elm app's sleepAndPoll interval exactly.
const POLL_INTERVAL_MS = 50;

export function usePolling() {
  const [state, setState] = useState<State | null>(null);
  const [error, setError] = useState<string | null>(null);
  const timerRef = useRef<number | null>(null);

  const applyState = useCallback((newState: State) => {
    setState(newState);
    setError(null);
    if (timerRef.current !== null) {
      window.clearTimeout(timerRef.current);
      timerRef.current = null;
    }
    if (newState.running) {
      timerRef.current = window.setTimeout(() => {
        getState().then(applyState).catch(handleError);
      }, POLL_INTERVAL_MS);
    }
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);

  const handleError = useCallback((err: unknown) => {
    setError(err instanceof Error ? err.message : String(err));
  }, []);

  useEffect(() => {
    getState().then(applyState).catch(handleError);
    return () => {
      if (timerRef.current !== null) {
        window.clearTimeout(timerRef.current);
      }
    };
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);

  return { state, error, applyState, setError: handleError };
}
```

Note: `applyState` is defined with `useCallback(..., [])` and calls itself recursively via the closed-over reference — this is safe because the function is only ever referenced through `timerRef`'s callback and the returned value, never redefined mid-flight, so there's no stale-closure risk despite referencing itself before its own declaration completes (a `useCallback` with an empty dependency array is created once and its identity is stable for the component's lifetime).

- [ ] **Step 2: Implement the color scale helper**

Create `frontend/src/colorScale.ts`:

```typescript
import { scaleDiverging } from 'd3-scale';
import { interpolateRdYlGn } from 'd3-scale-chromatic';

// Builds a factory cash -> color function, domain-fit to the given
// population's min/max/zero each time it's called, so the gradient stays
// meaningful as the economy's scale changes over a run rather than
// saturating to a single color against a fixed threshold. Diverges at 0
// (the insolvency boundary): red below, green above.
export function cashColorScale(cashValues: number[]): (cash: number) => string {
  if (cashValues.length === 0) {
    return () => 'grey';
  }
  const min = Math.min(...cashValues, -1);
  const max = Math.max(...cashValues, 1);
  const scale = scaleDiverging(interpolateRdYlGn).domain([min, 0, max]);
  return (cash: number) => scale(cash);
}
```

(`Math.min(...cashValues, -1)` / `Math.max(...cashValues, 1)` guard against a degenerate domain when every factory happens to have exactly the same sign or value, keeping `scaleDiverging`'s three-stop domain strictly ordered.)

```tsx
import { type PropsWithChildren, useEffect, useRef } from 'react';
import { select } from 'd3-selection';
import { zoom, zoomIdentity, type D3ZoomEvent } from 'd3-zoom';
import type { Bounds } from '../types';

// World-space tile layout, matching the original CustomSvg.elm constants
// exactly: a 5x5 grid of 32000-unit tiles with its corner at (0, -160300).
const MAP_CORNER_X = 0;
const MAP_CORNER_Y = -160300;
const MAP_TILE_SIZE = 32000;
const MAP_GRID_SIZE = 5;

interface MapViewProps {
  bounds: Bounds;
}

export default function MapView({ bounds, children }: PropsWithChildren<MapViewProps>) {
  const svgRef = useRef<SVGSVGElement | null>(null);
  const zoomGroupRef = useRef<SVGGElement | null>(null);

  useEffect(() => {
    if (!svgRef.current || !zoomGroupRef.current) {
      return;
    }
    const svgSelection = select(svgRef.current);
    const zoomGroupSelection = select(zoomGroupRef.current);
    const zoomBehavior = zoom<SVGSVGElement, unknown>()
      .scaleExtent([0.05, 20])
      .on('zoom', (event: D3ZoomEvent<SVGSVGElement, unknown>) => {
        zoomGroupSelection.attr('transform', event.transform.toString());
      });
    svgSelection.call(zoomBehavior);
    // Center roughly on the world bounds on first mount.
    const initialScale = 0.3;
    svgSelection.call(
      zoomBehavior.transform,
      zoomIdentity
        .translate(400, 300)
        .scale(initialScale)
        .translate(-(bounds.xmin + bounds.xmax) / 2, -(bounds.ymin + bounds.ymax) / 2),
    );
    return () => {
      svgSelection.on('.zoom', null);
    };
    // Only set up zoom once on mount -- bounds don't change after init.
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, []);

  const tiles = [];
  for (let x = 0; x < MAP_GRID_SIZE; x++) {
    for (let y = 0; y < MAP_GRID_SIZE; y++) {
      tiles.push(
        <image
          key={`${x}_${y}`}
          href={`/${x}_${y}.png`}
          x={MAP_CORNER_X + x * MAP_TILE_SIZE}
          y={MAP_CORNER_Y + y * MAP_TILE_SIZE}
          width={MAP_TILE_SIZE}
          height={MAP_TILE_SIZE}
        />,
      );
    }
  }

  return (
    <svg ref={svgRef} width="100%" height="100%" style={{ display: 'block', background: '#222' }}>
      <g ref={zoomGroupRef}>
        <g>{tiles}</g>
        {children}
      </g>
    </svg>
  );
}
```

(The initial `translate(400, 300)` is a rough starting screen-space offset so the map is visible on load rather than off in a corner — this and `initialScale` are starting points to tune visually once actually looked at in a browser, not exact values to defend.)

Create `frontend/src/components/NavRight.tsx`:

```tsx
import type { CSSProperties } from 'react';
import type { Recipe, State } from '../types';

interface NavRightProps {
  state: State;
  recipes: Recipe[];
  onSetRecipe: (name: string, active: boolean) => void;
}

const navColumnStyle: CSSProperties = {
  width: 200,
  padding: 4,
  background: '#808080',
  border: '2px solid black',
  display: 'flex',
  flexDirection: 'column',
  gap: 4,
  overflowY: 'auto',
};

const itemStyle: CSSProperties = {
  background: 'white',
  border: '1px solid black',
  padding: 4,
};

export default function NavRight({ state, recipes, onSetRecipe }: NavRightProps) {
  const alternates = recipes.filter((r) => r.name.startsWith('Alternate:')).sort((a, b) => a.name.localeCompare(b.name));
  const normal = recipes.filter((r) => !r.name.startsWith('Alternate:')).sort((a, b) => a.name.localeCompare(b.name));
  const inactiveResources = state.resources.filter((r) => !r.active).length;

  return (
    <div style={navColumnStyle}>
      <div style={itemStyle}>Resources: {state.resources.length}</div>
      <div style={itemStyle}>Factories: {state.factories.length}</div>
      <div style={itemStyle}>Sinks: {state.sinks.length}</div>
      <div style={itemStyle}>Transports: {state.transports.length}</div>
      <div style={itemStyle}>Inactive: {inactiveResources}</div>
      {[...alternates, ...normal].map((recipe) => (
        <label key={recipe.name} style={{ ...itemStyle, display: 'flex', alignItems: 'center', gap: 4, fontSize: 11 }}>
          <input
            type="checkbox"
            checked={recipe.active}
            onChange={(e) => onSetRecipe(recipe.name, e.target.checked)}
          />
          {recipe.name}
        </label>
      ))}
    </div>
  );
}
```

```tsx
import { useEffect, useState } from 'react';
import { usePolling } from './hooks/usePolling';
import MapView from './components/MapView';

function App() {
```
